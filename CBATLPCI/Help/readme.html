Improvements and Changes in MBSA 2.3

MBSA 2.3 includes the most popular scanning abilities of previous versions and adds these additional features:

Additional Features:

  • Ability to choose offline mode from graphical and command-line interfaces (/offline option)
  • Support for additional security catalogs (for future use)
  • Command-line option to obtain catalogs from user-selected directory or network share using /cabpath option
  • Compatibility with WSUS 3.0 technologies and the latest Windows Update Agent features
  • All vulnerability assessment (VA) checks have been extended to work on x64 (64-bit) platforms
  • Updated graphical user interface
  • Command-line option to redirect reports to a user-selected directory or network share using /rd option
  • Consistency with Windows Update, Microsoft Update, Windows Server Update Services (WSUS) server, Systems Management Server (SMS) 2003 and System Center Configuration Manager (SCCM)
  • Access to the live Microsoft Update site for published content for live (online) security update assessment as well as an offline catalog for customers with limited or secure internet access.
  • Automatic determination of the client's assigned WSUS server, eliminating the need to specify the client's WSUS server before scanning to ensure the correct assigned WSUS server will be used
  • Ability to provide a "not yet approved" score for unapproved WSUS updates
  • Ability to run multiple copies of MBSA for increased scanning performance
  • Current Update Compliance appears in the report; installed and needed updates are reported together in a single scan report
  • Updates that require a restart of the computer will be annotated as the reason for non-compliance both in the GUI and command-line interface
  • Accepts an input file of computers and IP addresses to be scanned
  • Reports can now be easily shared and viewed
  • Structured XML output offers simplified integration for update scanning
  • Ability to pass a specific username and password on the command line for both update scanning and security misconfiguration scanning
  • Option to automatically configure clients with prerequisites to ensure the latest required version of Windows Update Agent needed for scanning
  • Specific Web links for locating updates and taking necessary actions
  • Bulletin severity ratings available in report details

Fixes and changes from previous versions:

  • Corrects automatic fallback to offline mode if Microsoft Update or WSUS servers are unavailable (corrects an MBSA-specific issue introduced with the newest Windows Update Agent)
  • Removed download link in completed scan reports since its no longer possible to accurately identify the correct package in a multi-package bundle, which is how the majority of updates are now released
  • Removed product version from cache directory path (now C:\Documents and Settings\<username>\Local Settings\Application Data\Microsoft\MBSA\Cache).

Localization:

  • MBSA releases are available for German, Japanese, and French.
  • The WSUSSCN2.CAB file is localized to all supported languages and will be automatically downloaded and used by the tool for any client language or locale being scanned. Results are stored in the report based on the MBSA installation language.

MBSA can scan for security updates in the following products:

  • All products supported by Microsoft Update including:
  • Windows XP, Windows Server 2003, Windows Vista, Windows Server 2008, Windows 7, Windows Server 2008 R2, Windows 8, Windows Server 2012, Windows 8.1, Windows Server 2012 R2 - including 64-bit (x64 and ia64) support
  • Microsoft Windows XP Embedded (limited to remote scanning and limited to local scanning via the /xmlout command line switch)
  • Microsoft Office XP and later
  • Exchange Server 2000 and later
  • All Windows components (such as MSXML, MDAC, Windows Media Player, Outlook Express/Windows Mail)
  • .Net Framework 1.0 and later
  • SQL Server 2000 with service pack 3 and later
  • Additional and new Microsoft products as they are added to Microsoft Update (see the MBSA home page for details)

Additional Configuration Checks:

  • Incomplete Updates check - (for recently released updates that require a computer restart to take effect)

Additional MBSA CLI Switches:

  • /offline - Forces offline mode; easier than using /wa (WSUS Approved) or /wi (WSUS Ignore) to force offline mode
  • /addonly - Use only additional catalogs (if available)
  • /noadd - Do not use additional catalogs (even if available)
  • /cabpath - Location of additional catalogs (if available)
  • /ia - (Attempt to install a newer version of Windows Update Agent if one is required for scanning)
  • /mu - (Force target clients to use the live Microsoft Update site to determine security compliance regardless of WSUS server assignment)
  • /nd - (Force target clients not to use files from the Microsoft Web site to determine security compliance; offline catalog will be used instead)
  • /rd - (Used to place completed scan reports in a user-selected directory or network share. Also to retrieve scan reports from a user-selected location)
  • /qt - (do not display the report output automatically after a single computer scan)

Replaced MBSA CLI Switches for MBSA 1.2.1 compatibility:

  • /wi, /wa replace the /sus option
  • /catalog replaces the /x option
  • /xmlout replaces the /HF option
  • /target replaces the /i, /c and /h options
  • /listfile replaces the /fh and /fip options
  • /ld replaces the /v option

MBSA CLI switches removed from previous versions:

  • /n - (ranges and computer lists can be specified by the /r and /listfile options respectively)
  • /e - (structured output in XML can be parsed from the reports)
  • /history - (current update compliance is included by default when scanning)
  • /fq - (structured output in XML can be parsed from the reports)
  • /s, /sum, /nosum and /z - (Automatic Updates and Microsoft Update catalog detection logic is always used)
  • /o tab | wrap - (structured output in XML can be parsed from the reports)
  • /t - (multiple copies of MBSA may now be used on the same scanning computer)
  • /f - (console redirection can be used to write additional output to a file)
  • /nai - (Do not install a newer version of the Windows Update Agent even if required for scanning - which is the new default setting. Use the /ia switch to automatically install the required Windows Update Agent)
  • /nm - (Do not configure computers to use the Microsoft Update site for scanning by default. /mu switch is required to use Microsoft Update site)

 

Dropped Features and Workarounds

Issue

Reasons why an update is considered missing are not provided in this version of MBSA, however when an update has been installed and a computer restart is required for it to take effect, this reason will be provided in the report.

Workaround

The built-in logging of the Automatic Updates service found in the WindowsUpdate.log file can be used to inspect the detection behavior for troubleshooting.

Issue

Installation history (explicitly installed / effectively installed) no longer reflect those updates that have been superseded by another update. The behavior of this feature follows the behavior for missing updates, where only the most recent non-superseded update is shown.

Workaround

Refer to the bulletin, fixlist, or TechNet search page to identify each previous bulletin that has been included in the update in question from the Current Update Compliance listing.

Issue

Support for SUS 1.0 does not exist in MBSA 2.

Workaround

Upgrade from SUS 1.0 to Windows Server Update Services.
