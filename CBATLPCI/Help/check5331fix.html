SQL Server on Domain Controller

Issue

We recommend that you do not run Microsoft® SQL Server™ on a domain controller. Domain controllers contain sensitive data such as user account information, and they should not be used in another role. If you run a SQL Server database on a domain controller, you increase the complexity involved in securing the server and protecting against attack.

Solution

We recommend that you run SQL Server and domain controllers on separate, dedicated computers.
